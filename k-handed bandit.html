Idea problemu¶

Wyobra≈∫my sobie, ≈ºe jeste≈õmy w kasynie i mamy przed sobƒÖ automaty do gry. Takie automaty sƒÖ popularnie nazywane "jednorƒôkimi bandytami". Z kt√≥rych automat√≥w korzystaƒá, by zmaksymalizowaƒá sw√≥j doch√≥d z wizyty w kasynie? Jako gracze nie wiemy, jakie jest prawdopodobie≈Ñstwo zdobycia nagrody dla poszczeg√≥lnych maszyn. Nie wiemy tak≈ºe, jaka nagroda czeka nas w przypadku zdobycia nagrody. O tym, jaka jest nasza nagroda, dowiadujemy siƒô dopiero po jej zdobyciu. Gdyby nagroda by≈Ça znana, problem by≈Çby banalny - ciƒÖgnƒôliby≈õmy za ramiƒô, gdzie nagroda bƒôdzie najwiƒôksza.

Jest to tzw. problem wielorƒôkiego bandyty (ang. multi-handed bandit). Opisuje on sytuacjƒô gry w kasynie (stƒÖd wziƒô≈Ça siƒô jego nazwa), ale nie tylko. Polega on zawsze na tym, ≈ºe mamy ≈õrodowisko, w kt√≥rym mamy do wyboru r√≥≈ºne opcje, natomiast nie wiemy, jakie nagrody mo≈ºe nam przynie≈õƒá wyb√≥r poszczeg√≥lnych opcji. Naszym celem jest zmaksymalizowanie nagrody w d≈Çugim horyzoncie czasowym poprzez odpowiednie wybieranie opcji.

Przyk≈Çadem problemu, kt√≥ry mo≈ºna sprowadziƒá do problemu wielorƒôkiego bandyty jest wyb√≥r strategii przy leczeniu choroby. Mamy do dyspozycji r√≥≈ºne akcje, kt√≥re mo≈ºemy zastosowaƒá podczas terapii i nie wiemy, kt√≥re z nich sƒÖ skuteczne. Sformalizowanie problemu¶

Chcemy znale≈∫ƒá, kt√≥ra akcja przynosi najwiƒôksze zyski (i w zwiƒÖzku z tym - kt√≥rej akcji nale≈ºy u≈ºywaƒá). Dobrym sposobem na oszacowanie warto≈õci akcji jest wyznaczenie ≈õredniej nagr√≥d z przesz≈Ço≈õci dla danej akcji. $$Q_t(a_i) = \frac{\sum_{k=0}^{t-1} R_k I_k^i}{\sum_{k=0}^{t-1} I_k^i}$$ gdzie

  1. $Q_t(a_i)$ to przewidytana warto≈õƒá nagrody, wyznaczona dla i-tego ramienia w czasie t
  2. $I_k^i$ = 1, gdy $k$-ta akcja by≈Ça akcjƒÖ $a_i$ lub 0 wpp
  3. $R_k$ to nagroda uzyskana w k-tej akcji.

Podstawowy algorytm¶

Opiszƒô podstawowƒÖ wersjƒô algorytmu. W dalszej czƒô≈õci pracy bƒôdƒô wprowadza≈Ç modyfikacje do podstawowego modelu.

Dla ka≈ºdego ramienia a:

  1. Q(a) = 0
  2. N(a) = 0 # jest to licznik pociƒÖgniƒôƒá dla ramienia

W pƒôtli wykonujemy nastƒôpujƒÖce kroki:

  1. Wybierz ramiƒô, kt√≥re ma najwiƒôksze Q (bazujemy tutaj na przekonaniu, ≈ºe je≈õli do tej pory ramiƒô dswa≈Ço najwiƒôksze nagrody, to warto za nie ciƒÖgnƒÖƒá w przysz≈Ço≈õci

  2. Zagraj za pomocƒÖ rƒôki j i sprawd≈∫, jakƒÖ nagrodƒô otrzymasz: $R = getRewardFromBandit(j)$

  3. Uaktualnij warto≈õƒá przewidywanej nagrody. Jest tutaj u≈ºywany pewien trik matematyczny (patrz Sutton, Barto: str 31)

$Q(a) = Q(a) + \frac{1}{N(a)} [R - Q(a)]$

Algorytm epsilon-zachłanny¶

Powy≈ºszy algorytm ma pewnƒÖ powa≈ºnƒÖ wadƒô. Gdy pociƒÖgniemy za wajchƒô i dostaniemy nagrodƒô, bƒôdziemy sƒÖdzili, ≈ºe jest ona najlepsza, bo da≈Ça nam najlepszy wynik (inne wajchy rzeczywi≈õcie nic nie da≈Çy, ale te≈º i nie mog≈Çy nic daƒá). Przy u≈ºyciu takiej zach≈Çannej strategii mo≈ºemy zatrzymaƒá siƒô w stanie, kt√≥ry daje najlepsze wyniki, ale tylko lokalnie. Byƒá mo≈ºe przy u≈ºyciu innej wajchy mogliby≈õmy zyskaƒá wiƒôcej.

Dlatego opr√≥cz eksploatacji, czyli czerpania zysk√≥w z dotychczas znanych rozwiƒÖza≈Ñ wa≈ºna jest tak≈ºe eksploracja, czyli szukanie rozwiƒÖza≈Ñ, kt√≥re mogƒÖ poprawiƒá nasze zyski d≈Çugofalowo.

Eksploracja polega na losowym wybraniu akcji innej ni≈º ta, kt√≥rej wyb√≥r m√≥g≈Çby byƒá sugerowany przez najwiƒôkszƒÖ warto≈õƒá oczekiwanej nagrody.

Wprowadzamy zatem wpsp√≥≈Çczynnik $\epsilon$ (r√≥wny zazwyczaj 0.1). Podczas gry, z prawdopodobie≈Ñstwem $\epsilon$ bƒôdziemy ciƒÖgnƒÖƒá za losowƒÖ wajchƒô.

Zatem w tym przypadku, krok algorytmu:

  1. Wybierz ramiƒô, kt√≥re ma najwiƒôksze Q (bazujemy tutaj na przekonaniu, ≈ºe je≈õli do tej pory ramiƒô dawa≈Ço najwiƒôksze nagrody, to warto za nie ciƒÖgnƒÖƒá w przysz≈Ço≈õci.

Zmieni siƒô na:

  1. Wybierz ramiƒô, kt√≥re ma najwiƒôksze Q lub losowo z prawdopodobie≈Ñstwem $\epsilon$ - inne ramiƒô

def playBandit(eps, steps, probabilities, fun = functions["lin"]):

    Q = np.zeros(k)
    N = np.zeros(k)

    Qs = []

    # Uruchamiamy ca≈Çy algorytm wiele razy zmieniajƒÖc ilo≈õƒá¬†krok√≥w w algorytmie. Suma nagr√≥d powinna siƒô zmieniaƒá.
    for step in range(1, steps):
        for i in range(step):
            # Wybierzmy j - indeks rƒôki, za kt√≥rƒÖ pociƒÖgniemy    
            rand = np.random.rand()
            if rand <= eps:
                j = np.random.randint(k)
            else:
                maxBandits = np.where(Q == np.max(Q))[0]
                j = random.choice(maxBandits)

            R = getRewardFromBandit(j, fun, probabilities)

            N[j] = N[j] + 1
            Q[j] = Q[j] + (1/N[j]) * (R - Q[j])

        # Dodajemy sumƒô nagr√≥d
        Qs.append(sum(Q))

    return (range(1, steps), Qs)

Problem wielorƒôkiego bandyty mo≈ºna rozszerzaƒá na wiele sposob√≥w. Opiszƒô i zaimplementujƒô dwa z nich.

Problem niestacjonarny¶

Do tej pory nie zak≈Çadali≈õmy, ≈ºe warto≈õci oczekiwane nagr√≥d (i rozk≈Çady tych nagr√≥d) mogƒÖ siƒô zmienaƒá podczas gry. By≈Ç to tak zwany problem stacjonarny.

W problemie niestacjonarnym rozk≈Çad mo≈ºe siƒô zmieniaƒá. Dlatego powinni≈õmy wprowadziƒá mechanizm, kt√≥ry pozwoli≈Çby na premiowanie niedawno wykonanych akcji (sƒÖ one najbardziej aktualne, wiƒôc i najbardziej miarodajne). U≈ºycie ≈õredniej arytmetycznej do obliczania warto≈õci stan√≥w jest z≈Çym pomys≈Çem, bo premiuje wszystkie akcje jednakowo.

Mo≈ºemy zmodyfikowaƒá wz√≥r s≈Çu≈ºƒÖcy do obliczania warto≈õci stan√≥w w spos√≥b nastƒôpujƒÖcy:

$$Q_{t+1}=\alpha R_t + (1 - \alpha)Q_t$$ gdzie $\alpha \in (0,1\rbrack$ - sta≈Ça okre≈õlajƒÖca wp≈Çyw najnowszej nagrody na oszacowanie akcji

def playBanditAlfa(eps, steps, probabilities, fun = functions["lin"], alfa = 0.2):

    Q = np.zeros(k)
    N = np.zeros(k)

    Qs = []

    # Uruchamiamy ca≈Çy algorytm wiele razy zmieniajƒÖc ilo≈õƒá krok√≥w w algorytmie. Suma nagr√≥d powinna siƒô zmieniaƒá.
    for step in range(1, steps):
        for i in range(step):
            # Wybierzmy j - indeks rƒôki, za kt√≥rƒÖ pociƒÖgniemy    
            rand = np.random.rand()
            if rand <= eps:
                j = np.random.randint(k)
            else:
                maxBandits = np.where(Q == np.max(Q))[0]
                j = random.choice(maxBandits)

            R = getRewardFromBandit(j, fun, probabilities)

            N[j] = N[j] + 1
            Q[j] = Q[j] * (1-alfa) + R * alfa

        # Dodajemy sumƒô nagr√≥d
        Qs.append(sum(Q))

    return (range(1, steps), Qs)

Inna interpretacja funkcji nagrody¶

W innej interpretacji problemu, warto≈õci prawdopodobie≈Ñstwa wystƒÖpienia dla ramion bierzemy tak, jak w opcji trzeciej, to znaczy: prawdopobobie≈Ñstwo r√≥≈ºne dla ka≈ºdego ramienia, prawdopodobie≈Ñstwo jest dystrybuantƒÖ w punkcie, bƒôdƒÖcym warto≈õciƒÖ oczekiwanƒÖ.

Zupe≈Çnie inna jest interpretacja warto≈õci funkcji nagrody. Wielko≈õƒá nagrody bƒôdzie zale≈ºa≈Ça tylko od ilo≈õci pociƒÖgniƒôƒá za ramiona. Funkcje nagrody nadal bƒôdƒÖ liniowe lub kwadratowe.

Mo≈ºna r√≥≈ºnie interpretowaƒá ilo≈õƒá pociƒÖgniƒôƒá za ramiona:

  1. Ilo≈õƒá u≈ºyƒá wszystkich ramion w sumie - licznik u≈ºyƒá jest globalny
  2. Ilo≈õƒá u≈ºyƒá poszeczg√≥lnych ramion - licznik jest per ramiono.
